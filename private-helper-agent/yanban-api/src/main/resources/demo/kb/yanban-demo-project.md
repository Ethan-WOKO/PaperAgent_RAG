# 研伴 ScholarAI 项目说明

研伴 ScholarAI 是一个面向研究生、科研助理和小型课题组的私有研究助手。它把知识库问答、文献检索、论文处理和多步骤 Agent 工作流放在同一个工作台里，目标是减少研究者在资料查找、证据整理和任务推进上的重复劳动。

## 项目主要解决的三类科研工作流问题

1. 私有资料难复用：实验记录、论文笔记、组会纪要和项目文档常分散在不同文件里。研伴通过上传解析、切块、embedding 和检索，把这些资料变成可问答的私有知识库。
2. 文献与证据整理耗时：研究者需要在网页、论文、笔记和知识库之间来回切换。研伴希望把知识库检索、网页搜索、文献搜索和回答生成串成一个可追踪的流程。
3. 复杂任务缺少执行轨迹：例如“完善 Agent 能力”不是一次问答能完成的任务。计划模式会先拆解目标，再记录步骤、工具结果和执行状态，方便回看和复盘。

## 当前演示可体验的能力

- 游客一键登录：面试官扫码进入 `/demo` 后，不需要知道账号密码。
- 私有知识库：系统预置项目说明、RAG 笔记和虚拟实验室日程，适合验证知识库问答。
- 流式聊天：普通回答优先走 WebSocket 流式输出，连接异常时前端会用普通 HTTP 发送兜底。
- Agent 计划模式：适合把两周迭代、调研任务、论文处理任务拆成步骤。
- 移动端入口：登录、Demo、聊天、知识库、论文和设置页面都按手机扫码访问做了基础适配。

## Demo 体验边界

游客账号是共享账号，每天凌晨会清理会话和用户上传内容。请不要上传真实 API Key、未公开论文原稿、导师联系方式或任何隐私资料。Demo 允许少量小文件上传，主要用于展示检索和问答链路，不用于长期保存数据。
